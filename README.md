## 🔍 Fluxo de Trabalho em QA

Minha abordagem para QA segue este fluxo:

```
1. ANÁLISE
   └─ Ler requisitos e entender funcionalidade

2. DESIGN
   └─ Criar casos de teste estruturados
   └─ Identificar cenários de sucesso e falha
   └─ Incluir casos extremos

3. EXECUÇÃO
   └─ Testar manualmente a aplicação
   └─ Documentar resultados
   └─ Capturar evidências (screenshots)

4. REPORTE
   └─ Documentar bugs encontrados
   └─ Preparar relatórios
   └─ Fornecer feedback ao time

5. VALIDAÇÃO
   └─ Revalidar correções
   └─ Executar testes de regressão
   └─ Confirmar encerramento de issues
```
